Как работает программа.

В классе Main инициируется выполнение.

В классе Test находятся методы с тестами лаб. работ и выполненные задания к ним в виде отдельных методов.

Класс Service описывает предоставляемую услугу.

Интерфейс Tariff объединяет несколько услуг в тариф и предоставляет методы для работы с ними.
Этот интерфейс реализуют классы EntityTariff (юр. лицо) и IndividualTariff (обычн. чел.).
Их отличие в том, что EntityTariff собирает в себе услуги в виде двусвязного списка (узел списка представлен классом Node),
а IndividualTariff в виде обычного массива.

Интерфейс Account нужен для создания классов, которые будут описывать учётную запись человека (его счёт) или организации.
В классе IndividualAccount хранятся сведение о человеке (его описывает класс Person) и его тариф. Класс EntityTariff вместо
ссылки на экземпляр класса Person хранит название организации name типа String.
У обоих типов аккаунта есть поле для индивидуального номера.

Класс AccountManager описывает массив аккаунтов и предоставляет методы для взаимодействия с ними.
